# ADR 005: Server Composition Root and Runtime Bootstrap Extraction

Status: Accepted

Context
- The server bootstrap and composition pattern evolves across crates; no formal ADR for the root composition exists.

Decision
- Define a canonical server composition root and a runtime bootstrap protocol:
  - Centralize startup ordering and wiring of services in a designated bootstrap module.
  - Expose a minimal, well-typed bootstrap API to downstream crates.
- Document lifecycle hooks for initialization, wiring, and shutdown to avoid brittle startup code.

Consequences
- Improves reliability and testability of startup/shutdown sequences; easier refactors.
- May require small refactors to expose bootstrap hooks and to adjust tests accordingly.
