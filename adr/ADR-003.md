# ADR 003: CLI Product-Line Strategy (Agent/CLI/Repl)

Status: Accepted

Context
- The project exposes multiple entry points: CLI agents, repl, and programmatic APIs.
- There is no explicit ADR documenting how the CLI product line is intended to evolve or how features are composed.

Decision
- Define a product-line strategy for the CLI surface: stable core, with a family of subcommands for agents, tooling, and interactive REPL.
- Establish versioning and compatibility policy for CLI changes; ensure backward compatibility for major features.
- Introduce a channel for feature flags to gate experimental CLI capabilities behind releases.

Consequences
- Provides clarity for future CLI expansion and feature gating.
- Requires documentation updates and potentially minor CLI refactors to align with the strategy.
