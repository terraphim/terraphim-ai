# ADR 002: Workspace Dependency Governance Policy

Status: Accepted

Context
- The repo contains multiple crates with inter-crate dependencies. Governance around workspace dependencies is informal.
- ADR 001 established layering; this ADR formalizes how workspace dependencies are resolved and version-locked.

Decision
- All workspace dependencies MUST be version-locked and tracked in a central dependency manifest (e.g., a workspace-level Cargo.toml or lockfile).
- Changes to inter-crate APIs MUST be reviewed via PRs with explicit compatibility notes and follow semantic versioning when public interfaces change.
- Introduce a policy: avoid circular dependencies; define a clear root of aggregate services.

Consequences
- Improves reproducibility and governance of builds; helps ensure stable, auditable dependency graphs.
- May require pinning versions more strictly and updating docs with compatibility notes.
