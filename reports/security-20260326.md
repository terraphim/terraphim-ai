## Critical Vulnerabilities

### CVE-001: aws-lc-sys X.509 Name Constraints Bypass (HIGH)
**RustSec IDs:** RUSTSEC-2026-0044, RUSTSEC-2026-0048  
**Package:** `aws-lc-sys` v0.38.0  
**Severity:** 7.4 HIGH  
**Solution:** Upgrade to >=0.39.0  
**Date:** 2026-03-19

**Impact:** Certificate validation bypass enabling man-in-the-middle attacks.

**Affected dependency chain:**
- aws-lc-sys -> aws-lc-rs -> salvo-acme -> salvo
- aws-lc-sys -> rustls -> reqwest, ureq, tokio-rustls
- Deep transitive dependency affecting TLS across entire application

---

### CVE-002: rustls-webpki CRL Validation Bypass (HIGH)
**RustSec ID:** RUSTSEC-2026-0049  
**Package:** `rustls-webpki` v0.102.8  
**Solution:** Upgrade to >=0.103.10  
**Date:** 2026-03-20

**Affected chain:**
- rustls-webpki 0.102.8 -> rustls 0.22.4 -> tungstenite -> tokio-tungstenite -> serenity -> terraphim_tinyclaw

## Remediation Actions

### Immediate (P0)

1. **Upgrade aws-lc-sys to >=0.39.0**
   ```bash
   cargo update -p aws-lc-sys
   cargo update -p aws-lc-rs
   ```

2. **Upgrade rustls-webpki to >=0.103.10**
   ```bash
   cargo update -p rustls-webpki
   # May require rustls version update
   ```

### Short-term (P1)

3. **Restrict Ollama to localhost**
   ```bash
   export OLLAMA_HOST=127.0.0.1:11434
   # Or systemd override
   ```

4. **Replace rustls-pemfile** with `rustls-pki-types`

### Medium-term (P2)

5. **Migrate from bincode 1.x** - Consider `bincode2` or `postcard`
6. **Replace fxhash** with `rustc-hash` or `ahash::AHasher`
7. **Replace trust-dns-proto** with `hickory-dns`
