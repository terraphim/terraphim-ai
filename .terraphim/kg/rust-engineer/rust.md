# Rust Engineer Knowledge Graph

## Ownership and Borrowing

Ownership is Rust's core memory management model ensuring memory safety without a garbage collector.

[synonyms::] borrow, lifetime, ownership, move, reference, borrow-checker

## Async Programming

Async programming in Rust uses futures and the async/await syntax for non-blocking execution.

[synonyms::] async, await, future, tokio, runtime, non-blocking

## Traits and Generics

Traits define shared behaviour; generics enable type abstraction.

[synonyms::] trait, generic, impl, bound, associated-type, where-clause

## Error Handling

Rust uses Result and Option types for explicit error handling.

[synonyms::] result, option, unwrap, match, error, propagate

## Unsafe Rust

Unsafe Rust allows raw pointer dereferencing and FFI when the compiler cannot guarantee safety.

[synonyms::] unsafe, raw-pointer, ffi, c-binding, transmute

## Macros

Macros provide metaprogramming capabilities for code generation.

[synonyms::] macro, macro-rules, procedural-macro, derive, quote

## Cargo and Crates

Cargo is Rust's build system and package manager; crates are reusable libraries.

[synonyms::] cargo, crate, crate-io, workspace, manifest, dependency

## Testing

Rust supports unit tests, integration tests, documentation tests, and benchmarks.

[synonyms::] test, benchmark, mock, assertion, coverage, doctest

## Concurrency

Concurrency in Rust leverages threads, channels, and atomics for parallel execution.

[synonyms::] thread, channel, mpsc, mutex, arc, atomic, parallel

## WebAssembly

WebAssembly enables running Rust code in browsers and other Wasm hosts.

[synonyms::] wasm, wasm-bindgen, web, browser, wasi

## Memory Safety

Memory safety guarantees prevent null pointer dereferences, use-after-free, and data races.

[synonyms::] memory-safety, dangling-pointer, use-after-free, data-race, sandbox

## Type System

Rust's type system enforces correctness at compile time through strong static typing.

[synonyms::] type, static-type, inference, pattern-matching, exhaustive
