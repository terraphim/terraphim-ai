# Balanced Routing

Balanced routing provides an optimal balance between cost, performance, and quality for everyday tasks. This scenario is designed for standard development workloads where you need reliable performance without the premium cost of high-end models or the limitations of budget options.

route:: openrouter, anthropic/claude-3.5-sonnet

Balanced routing is used for:
- General development tasks and coding assistance
- Standard debugging and problem-solving
- Everyday documentation and code reviews
- Regular chat interactions and queries
- Typical AI-assisted workflows

Characteristics:
- Uses proven Claude 3.5 Sonnet model for reliable performance
- Provides excellent cost-to-performance ratio
- Suitable for most day-to-day development tasks
- Balances response quality with reasonable latency
- Optimized for frequent, routine interactions

Use cases:
- Code explanation and documentation generation
- Standard debugging and troubleshooting
- General knowledge queries and research
- Routine code reviews and suggestions
- Regular development workflow assistance
