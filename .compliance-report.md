### Issue 2.1: Critical Advisory Suppressed (MEDIUM)

```
RUSTSEC-2026-0049: rustls-webpki CRL revocation bypass
```

**Severity**: MEDIUM (but flagged for attention)
**Status**: Suppressed in deny.toml

**Details**:
- **Vector**: CRL (Certificate Revocation List) revocation check bypass in rustls-webpki 0.102.x
- **Affected**: serenity 0.12 → hyper-rustls 0.24 → rustls 0.21.x
- **Impact**: Discord integration (tinyclaw feature) could accept revoked certificates
- **Workaround**: Discord feature disabled by default in tinyclaw
- **Blocker**: serenity 0.13+ needed (not released yet as of 2026-04-07)

**Remediation**:
1. Verify Discord/tinyclaw is **not enabled in default features** of critical binaries
2. Document this dependency in security advisories
3. Monitor serenity releases for 0.13+ with rustls 0.23+ support
4. Pin serenity < 0.13 explicitly to prevent accidental upgrade

**Action**: Add comment to deny.toml linking to RUSTSEC tracking issue.

## 3. GDPR and Data Handling Patterns

### Data Protection Summary

**Status**: ADEQUATE - Appropriate patterns implemented

### Redaction Implementation (COMPLIANT)

Location: `crates/terraphim_agent/src/learnings/redaction.rs`

**Secrets automatically redacted**:
- AWS Access Keys: `AKIA[A-Z0-9]{16}`
- AWS Secret Keys: 40-char base64
- OpenAI API keys: `sk-*`
- Slack tokens: `xox*`
- GitHub PATs: `ghp_*`, `gho_*`
- Database connection strings: PostgreSQL, MySQL, MongoDB, Redis
- Environment variables: DATABASE_URL, API_KEY, PASSWORD, TOKEN, etc.

**Assessment**: Comprehensive pattern coverage. Redaction happens **before persistence** and **before logging**.

### Secret Storage (COMPLIANT)

Location: `crates/terraphim_atomic_client/src/auth.rs` and `crates/terraphim_onepassword_cli/src/lib.rs`

**Implementation**:
- 1Password integration for secret storage
- No hardcoded secrets in code
- Environment variable injection via `op inject`
- Secrets stripped before logging

**Assessment**: Follows principle of least privilege. Secrets never stored in application data.

### Logging/PII Patterns

Location: `crates/terraphim_agent/src/learnings/capture.rs`

**Behavior**:
- Captures failed commands for learning
- Applies redaction before capture
- No PII in logs by default

**Assessment**: Adequate. Should document what "redacted" PII is captured in learnings system.

### Quickwit Integration (LOG DATA)

Location: `crates/terraphim_middleware/src/haystack/quickwit.rs`

**Data Type**: Log and observability data (typically not PII, but organization-sensitive)

**Handling**:
- Authentication: Bearer token and Basic Auth support
- Credentials stored in `Haystack` config (uses 1Password for secrets)
- No client-side decryption or PII inference
- Acts as proxy to Quickwit server

**Assessment**: Adequate for log data. Must document data classification.
