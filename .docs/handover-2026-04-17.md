## Key Lessons Learned

1. **Rust version skew between local and CI**: CI uses `dtolnay/rust-toolchain@stable` (currently 1.95). Local dev may be on 1.94.1. New clippy lints in 1.95 do not appear locally, causing a whack-a-mole pattern where each CI run reveals a new batch. **Mitigation**: When fixing clippy lints, search the ENTIRE workspace proactively with `cargo clippy --workspace --all-targets` rather than waiting for CI.

2. **Pre-existing `lint-and-format` CI failure**: The `lint-and-format` job in `ci-native.yml` has a pre-existing failure -- `fff-search`'s `build.rs` detects CI and panics without the `zlob` feature. This is NOT caused by our changes. **Do NOT attempt to fix this without understanding the full context.**

3. **`sort_by` lint fix pattern**: For Copy types, use `sort_by_key(|x| std::cmp::Reverse(x.field))`. For non-Copy or complex closures where the borrow checker fights back, use `#[allow(clippy::unnecessary_sort_by)]` as a pragmatic escape hatch.

4. **Squash merge strategy**: PR #820 was squash-merged, collapsing 20+ commits into one. The individual clippy fix commits are visible in the PR body but not in main's history.
