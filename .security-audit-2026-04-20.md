## Dependency Vulnerabilities

### CVE-1: RUSTSEC-2026-0098 + RUSTSEC-2026-0099
**TLS Certificate Validation Bypass (Name Constraints)**
- **Affected Package:** rustls-webpki 0.101.7 / 0.102.8
- **Severity:** CRITICAL
- **Type:** Name constraints validation bypass for URI and wildcard certificates
- **Impact:** TLS clients can be tricked into accepting certificates from arbitrary domains
- **Attack Vector:** Network (remote)
- **Fix Available:** Upgrade to >=0.103.12 (requires rustls >=0.23.x)
- **Current Status:** Unresolved - downstream dependency chain blocks upgrade

**Dependency Chain:**
```
rustls-webpki 0.101.7 → rustls 0.21.12 → tokio-rustls 0.24.1 → reqwest 0.11.27
    ↓ (used in) terraphim_middleware → terraphim_service → CRITICAL CORE PATH
```

**Root Cause:** 
- `rustls 0.21.x` pins `rustls-webpki ^0.101` (incompatible with 0.103+)
- Upstream `reqwest 0.11.27` does not support rustls 0.23.x
- Requires major version bump of both reqwest (0.11 → 0.12) and tokio-rustls

### CVE-2: RUSTSEC-2026-0098 (Second Instance)
**TLS Certificate Validation Bypass (Name Constraints)**
- **Affected Package:** rustls-webpki 0.102.8
- **Severity:** CRITICAL
- **Status:** Unresolved (duplicate via rustls 0.22.4 chain)

**Dependency Chain:**
```
rustls-webpki 0.102.8 → rustls 0.22.4 → (serenity 0.12.5 dependency)
    ↓ (used in) terraphim_tinyclaw
```

**Root Cause:** 
- `serenity 0.12.5` pins `rustls 0.22.x` 
- No upgrade path available for serenity (Discord integration dependency)
- **Architectural Decision Required:** Accept CVE / Remove Discord features / Fork serenity

### CVE-3: RUSTSEC-2026-0097
**Rand Unsoundness with Custom Logger**
- **Affected Package:** rand 0.8.5 / 0.10.0
- **Severity:** HIGH
- **Type:** Unsound Rust code (UB potential)
- **Impact:** Undefined behavior if custom logger used with `rand::rng()`
- **Fix Available:** Upgrade to rand 0.9.0+
- **Current Status:** Unresolved

**Dependency Chain:**
```
rand 0.8.5 → tungstenite 0.21.0 → tokio-tungstenite 0.21.0 → serenity 0.12.5
    ↓ terraphim_tinyclaw (also via sqlx → opendal)
```
