## What Worked

- `assert_cmd::cargo_bin` is the correct pattern — uses `env!("CARGO_BIN_EXE_<name>")`, no nested cargo
- Synthetic git repos via `tempfile::tempdir()` + `std::process::Command::new("git")` are real, not mocked
- In-process `Arc<Mutex<T>>` concurrency tests are fully deterministic; no `#[serial]` guard needed
- Explicit `drop(repo_dir)` at end of test documents intent clearly

## What to Avoid

- Never use `git rev-list --max-parents=0 HEAD` in tests — fails in shallow CI clones
- Never use `cargo run` inside `cargo test --workspace` integration tests — artifact lock contention
- Never use concurrent subprocess writes to SQLite in tests — SQLITE_BUSY is non-deterministic
