## Task Description

`FirecrackerExecutor::validate()` in `crates/terraphim_rlm/src/executor/firecracker.rs:500`
is a stub that always returns `ValidationResult::valid(Vec::new())`. The KG-based safety
contract requires it to use `terraphim_automata` (via `KnowledgeGraphValidator`) to check
the input string against the role's knowledge graph.

## Subtasks

1. [S] Add `validator: KnowledgeGraphValidator` field to `FirecrackerExecutor` struct (cfg-gated)
2. [S] Initialise validator from `config.kg_strictness` + `config.kg_max_retries` in `new()`
3. [S] Add `with_validator()` builder method for test injection
4. [M] Implement `validate()` -- delegate to `kg_validate()` helper with cfg gate, convert result types
5. [S] Add unit tests: disabled validator passes, empty input passes, normal mode no-thesaurus passes

## Key Files

- `crates/terraphim_rlm/src/executor/firecracker.rs` -- primary change

## Key Decisions

- Use `KnowledgeGraphValidator::disabled()` default (no thesaurus = permissive), callers
  inject real thesaurus via `with_validator()`
- Two cfg-gated `kg_validate()` overloads (enabled/disabled) keep `validate()` clean
- `validator::ValidationResult.passed` maps to `context::ValidationResult.is_valid`
- Tests do NOT need KVM -- they test the KG conversion logic directly
