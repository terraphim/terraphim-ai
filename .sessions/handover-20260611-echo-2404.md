### Design Decisions

- **Fail-open**: if `list_pr_files()` returns an API error, contamination check is skipped with `warn!` — clean PRs never get blocked by a transient API failure
- **Only "added" + "renamed" trigger gate**: modifying or removing an artefact file is fine
- **50-file limit**: `?limit=50` on Gitea PR files endpoint; artefact contamination surfaces in first few files of affected PRs
- **ARTEFACT_DENYLIST patterns**: `.pr_review_temp/`, `cursor_new.rs`, `reports/`, `.sessions/`, `.bak`, `.jsonl`, `EOF`, `PYEOF`
- **Verdict string carries paths**: `Hold("artefact contamination: <paths>")` — paths visible in `pr.evaluated` JSON log via existing `jsonlog::emit`
