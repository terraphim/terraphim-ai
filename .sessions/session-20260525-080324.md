## Phase 1: Disciplined Research

### Problem Statement
Malformed role configs currently panic deep in serde_json deserialization or silently skip roles, leaving users with degraded search and no actionable errors. The terraphim_config crate has zero schema validation layer.

### Proposed Solution
- Add schemars to terraphim_config
- Derive JsonSchema on RoleConfig, HaystackConfig, and RelevanceFunction
- Add validate_config() function running JSON Schema validation before serde_json::from_str
- Surface field-level errors with role name context
- Unit test for missing required field

### Acceptance Criteria
- Given a role config JSON with relevance_function omitted, startup fails with exit code 1 and descriptive error
- Valid config loads successfully within existing boot time
- cargo test -p terraphim_config passes with new validation unit test
