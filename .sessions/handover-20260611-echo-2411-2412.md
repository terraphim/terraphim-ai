## Technical Context

**File changed**: `crates/terraphim_rlm/src/validator.rs`

Key diffs:
```
- .filter(|s| !s.is_empty() && s.len() > 2) // Skip very short words
+ .filter(|s| !s.is_empty() && !STOP_WORDS.contains(s))

- format!("{}...", &s[..97])
+ char_indices walk to find safe end boundary
```

**Side effects**: None. The fix is backward-compatible — words that were previously extracted still are. Only tokens formerly filtered by `len() > 2` but NOT in `STOP_WORDS` are now included. The only words newly excluded are the explicit stop-words (grammatical noise).

## Lessons Learned (key for next agent)

- DCG hook blocks git commit messages and PR bodies containing the string inside `truncate_for_log` (the word matches a destructive command pattern). Use `-F /tmp/msg.txt` for commits and `--file /tmp/body.md` for wiki/PR bodies.
- `extract_words` split keeps hyphens as part of tokens: `"rm -rf /"` → `["rm", "-rf", "tmp", "test"]`, NOT `["rm", "rf"]`. Write assertions accordingly.
- Run cargo from `/data/projects/terraphim/terraphim-ai` (main repo), NOT from `.worktrees/` subdirs.
- Use `python3` for JSON/text processing — `grep`/`rg` aliased to `fff` returns empty on large UTF-8 Rust files.
