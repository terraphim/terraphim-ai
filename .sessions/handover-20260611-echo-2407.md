## Completed This Session

### Issue #2407 — Implement KG validation in FirecrackerExecutor::validate()

The stub `validate()` in `crates/terraphim_rlm/src/executor/firecracker.rs` that always
returned `ValidationResult::valid(Vec::new())` has been replaced with a real implementation.

**Files changed:**
- `crates/terraphim_rlm/src/executor/firecracker.rs` (+152 lines)
- `crates/terraphim_rlm/src/validator.rs` (+77 lines)

**What was done:**
1. Added `validator: KnowledgeGraphValidator` field to `FirecrackerExecutor` struct,
   gated by `#[cfg(feature = "kg-validation")]`. Initialised from config's
   `kg_strictness` and `kg_max_retries` without a thesaurus (permissive default).
2. Added `with_validator()` builder method for injecting a thesaurus-backed validator.
3. Added `kg_validate()` helper with two cfg-gated overloads:
   - With `kg-validation`: delegates to `KnowledgeGraphValidator`, converts
     `validator::ValidationResult` → `context::ValidationResult`
   - Without `kg-validation`: no-op returning valid
4. Wired `validate()` to call `kg_validate()`.
5. Added 4 unit tests in `validator::tests::with_thesaurus` using real
   `terraphim_types::Thesaurus` to verify the contract. All 127 tests pass.

**Key design decisions:**
- `validator::ValidationResult.passed` → `context::ValidationResult.is_valid`
- `unmatched_words` → `unknown_terms`
- Tests placed in `validator.rs` NOT `firecracker.rs` — the `firecracker` feature
  is absent from `full`/`default` and `fcctl-core` is commented out, so tests in
  `firecracker.rs` would never compile in CI
