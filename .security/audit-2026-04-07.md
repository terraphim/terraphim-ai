## Critical Findings

### [CRITICAL] RUSTSEC-2026-0049: TLS Certificate Revocation Bypass

**CVE Severity**: Privilege Escalation
**Affected Package**: `rustls-webpki 0.102.8`
**Patched Version**: `0.103.10+`

#### Vulnerability Details
The rustls-webpki CRL (Certificate Revocation List) validation logic contains a flaw where:
- If a certificate has multiple `distributionPoint` entries, only the **first** is checked against each CRL's `IssuingDistributionPoint`
- Subsequent `distributionPoint` entries are ignored
- With `UnknownStatusPolicy::Allow`, revoked certificates may be incorrectly accepted

#### Attack Scenario
1. CA is compromised or negligent
2. CRL properly lists revoked certificate under secondary distributionPoint
3. Primary distributionPoint is empty or missing from CRL
4. System accepts the "unrevoked" certificate despite actual revocation
5. Attacker continues using compromised credential

#### Root Cause Analysis
```
Dependency Chain:
terraphim-ai
  ↓ uses (optional) via serenity feature
serenity 0.12
  ↓ depends on
tokio-tungstenite 0.21
  ↓ depends on
rustls 0.22.4
  ↓ depends on
rustls-webpki 0.102.8  ← VULNERABLE
```

**Context**: serenity is used for Discord integration (optional feature, disabled by default)

**Recommendation**: Proceed with Option A (upgrade). The CRL bypass is fundamental to TLS security.

## High Severity Findings

### [HIGH] Network Exposure: LLM Provider API (Port 3456)

**Service**: terraphim-llm-provider (PID 947)
**Binding**: `0.0.0.0:3456` (fully exposed)
**Status**: Accessible from any network interface

#### Risk Assessment
- No authentication on LLM API
- Accepts natural language queries
- Could be exploited for:
  - Model inference abuse (resource exhaustion)
  - Information extraction from context
  - Denial of service

#### Remediation
```bash
# Bind to localhost only
# In service configuration or systemd socket:
ListenStream=127.0.0.1:3456
```

---

### [HIGH] Network Exposure: Ollama API (Port 11434)

**Service**: Ollama (local model server)
**Binding**: `0.0.0.0:11434` (fully exposed)
**Status**: Accessible from any network interface

#### Risk Assessment
- Unencrypted model API exposure
- No authentication
- Could download/execute arbitrary models
- Resource exhaustion risk

#### Remediation
```bash
# Bind to localhost only in ollama configuration
OLLAMA_HOST=127.0.0.1:11434

# OR add reverse proxy with authentication in front
```
