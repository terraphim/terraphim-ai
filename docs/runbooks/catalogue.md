# Canonical Full Skills Catalogue

## Source
Generated by `scripts/skill-installer-catalogue.sh` from one of three sources, in priority order:

1. **`tsm list`** (live, requires `TSM_TOKEN` or `~/.terraphim/token`)
2. **`~/.terraphim/registry/skills/`** (local clone from `tsm list` execution, no auth)
3. **Built-in fallback** (sentinel skills only)

The current committed `skills-catalogue.json` was generated from the local registry clone and contains 43 skills.

## Format
```json
{
  "source": "tsm|registry|fallback",
  "generated_at": "2026-06-14T07:50:13Z",
  "skill_count": 43,
  "skills": [
    { "name": "acceptance-testing", "auth_gated": true },
    { "name": "adf-orchestrate", "auth_gated": true },
    ...
  ],
  "expected_clis": ["claude-code", "codex", "pi", "pi-rust", "grok"],
  "notes": "..."
}
```

## Important Constraints

- **All 43 skills are auth-gated** in the current registry. They require `TSM_TOKEN` or `~/.terraphim/token` to install via marketplace.
- **Registry skills have `SKILL.md` but no `skill.toml`**. `tsm verify` requires `skill.toml` metadata which is only present for skills installed via the marketplace.
- **`tsm list` currently returns 0 skills without auth** even though the local registry has 43. This is because the marketplace API requires subscription-tier auth.

## Regenerate
```bash
TERRAPHIM_SKILLS_CATALOGUE_OUT=docs/runbooks/skills-catalogue.json \
TERRAPHIM_SKILLS_MANAGER_BIN=$(which tsm) \
  scripts/skill-installer-catalogue.sh
```
